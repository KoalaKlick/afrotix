# Event Member Identity & Voting/Attendance Flow

## Overview

This document describes the new approach for handling event participation (voting and attendance) on Afrotix. The system now uses "Event Members" with unique codes, replacing the previous reliance on organization-registered members.

---

## 1. Event Members

- **Definition:**
  - Event Members are individuals associated with a specific event for the purpose of voting or attendance.
  - They do not require a platform or organization account.

- **Attributes:**
  - Name
  - Email or Phone
  - Unique Code (generated per event)
  - Status (invited, attended, voted, etc.)

---

## 2. Workflow

### Event Setup
- Admin creates an event.
- Admin adds Event Members (individually or in bulk).
- System generates a unique code for each Event Member.
- Codes are sent to Event Members via nodemailer (email/SMS).

### Voting/Attendance
- Event Members visit the event page.
- They enter their unique code to vote or mark attendance.
- The system validates the code and records the action.
- Codes can be single-use or reusable per event rules.

---

## 3. API Endpoints

- `POST /events/:eventId/members` — Add event members and generate codes.
- `POST /events/:eventId/members/send-codes` — Send codes to event members via Mailaroo.
- `POST /events/:eventId/vote` — Submit a vote using a unique code.
- `POST /events/:eventId/attend` — Mark attendance using a unique code.

---

## 4. Data Model

**EventMember**
- id
- eventId (FK)
- name
- email/phone
- uniqueCode
- status (invited, attended, voted, etc.)
- createdAt

---

## 5. Key Points

- Only Event Members with valid codes can vote or attend events.
- No platform or organization login/account is required for Event Members.
- All code delivery is handled via Mailaroo.
- This system is separate from organization management (admin/member/owner roles).

---

## 6. Admin Features

- Add/manage Event Members for each event.
- Trigger code generation and sending.
- Track voting/attendance status per Event Member.

---

## 7. Security & Integrity

- Unique codes prevent duplicate voting/attendance.
- Codes can be reset/regenerated by admins if needed.
- All actions are logged for audit purposes.

---

## 8. Migration Notes

- Existing organization member voting/attendance logic should be deprecated in favor of this new flow.
- Ensure all event participation features reference Event Members and their codes.

---

## 9. Future Considerations

- Support for bulk import/export of Event Members.
- Optional notifications/reminders for Event Members.
- Analytics on participation rates.
